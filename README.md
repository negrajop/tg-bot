# VK Mood Tracker

Бот для ВКонтакте, который отслеживает настроение пользователей, хранит историю в SQLite и строит персональные графики с индексом счастья.

## Возможности

- отдельное хранение данных для каждого пользователя по `user_id`
- сохранение `username` или имени пользователя вместе с записью
- одна запись на пользователя в день с возможностью редактирования
- команда `/start` с текущим индексом счастья
- команда `/edit YYYY-MM-DD 4 YYYY-MM-DD 5` для массового редактирования
- кнопки настроения в клавиатуре VK
- персональный график настроения и индекса счастья
- генерация тестовых данных

## Стек

- Python
- vkbottle 4
- SQLite
- matplotlib
- python-dotenv

## Структура проекта

- `bot.py` - VK-бот на `vkbottle`
- `db.py` - работа с БД и миграция схемы
- `charts.py` - расчёт индекса счастья и генерация графиков
- `client.py` - сохранение графика пользователя напрямую из БД
- `seed_fake_data.py` - генерация тестовых пользовательских данных

## Подготовка

1. Создай виртуальное окружение и установи зависимости:

```powershell
python -m venv .venv
.\.venv\Scripts\pip install -r requirements.txt
```

2. Создай `.env` на основе `.env.example`.

Пример:

```env
VK_BOT_TOKEN=vk1.a.your_group_token
DB_PATH=database.db
```

`VK_BOT_TOKEN` - это токен сообщества VK. Его можно получить в настройках сообщества:
"Управление -> Работа с API -> Ключи доступа".

3. Включи Long Poll API для сообщества и разреши событие новых сообщений.

## Запуск бота

```powershell
.\.venv\Scripts\python.exe bot.py
```

Что умеет бот:

- `/start` или `Начать` - показывает текущий индекс счастья, клавиатуру и график
- кнопки настроения - сохраняют оценку за текущий день
- `/edit YYYY-MM-DD 4 YYYY-MM-DD 5` - редактирует сразу несколько дней
- текст `Отлично`, `Хорошо`, `Нормально`, `Плохо`, `Ужасно` - тоже сохраняет настроение

## Systemd

Для Linux-сервера есть готовые скрипты:

- `deploy/install_systemd.sh` - создаёт и запускает `systemd`-службу
- `deploy/uninstall_systemd.sh` - останавливает и удаляет службу

Установка:

```bash
sudo bash deploy/install_systemd.sh
```

Удаление:

```bash
sudo bash deploy/uninstall_systemd.sh
```

По умолчанию служба называется `tg-bot`, запускается из текущей директории проекта и использует:

- `/opt/tg-bot`
- `/opt/tg-bot/.venv/bin/python3`
- `/opt/tg-bot/.env`
- `/opt/tg-bot/bot.py`

При необходимости можно переопределить параметры через переменные окружения:

```bash
sudo SERVICE_NAME=vk-mood-bot APP_USER=root WORKDIR=/opt/tg-bot bash deploy/install_systemd.sh
```

Пример:

```text
/edit 2026-03-20 4 2026-03-21 5 2026-03-22 3
```

## Построение графика

Сохранить график конкретного пользователя:

```powershell
.\.venv\Scripts\python.exe client.py --user-id 42
```

По умолчанию график сохраняется в `mood_chart.png`.

Можно указать другой путь:

```powershell
.\.venv\Scripts\python.exe client.py --user-id 42 --output charts\alice.png
```

## Заполнение тестовыми данными

Скрипт создаёт или обновляет данные пользователя за период с `2026-03-01` по текущую дату системы.

```powershell
.\.venv\Scripts\python.exe seed_fake_data.py --user-id 42 --username demo_user
```

## Индекс счастья

Индекс счастья считается по последним 7 записям пользователя:

- `1` соответствует низкому уровню настроения
- `5` соответствует максимальному уровню
- итоговое значение масштабируется в диапазон от `0` до `100`

## Примечания

- в таблице используется ключ `(user_id, entry_date)`, поэтому запись за день обновляется, а не дублируется
- если раньше база была в старом формате, старая таблица будет сохранена как `mood_legacy_...`, а новая схема создастся автоматически
- не запускай несколько экземпляров `bot.py` одновременно с одним и тем же токеном сообщества
